在Docker中运行Hadoop+Spark集群

作者:米叔    发布于: 2017-07-19

大数据环境搭建一直是令初学者头疼的事情，提高了大数据学习入门的成本。环境搭建主要有单机模式、伪分布式模式、完全分布式模式。三种环境搭建成本高，换机器的话还要重新配置，不仅麻烦，还容易出错。Docker是一种容器，能够在不同的机器上很方便的运行统一的环境。于是米叔基于kiwenlau/hadoop:1.0（一个已经开发好的完全分布式Hadoop in Docker环境）开发了运行Hadoop+Spark的分布式Docker镜像，hadoop配置请参考hadoop-cluster-docker。

3节点集群


拉取Docker镜像
docker pull miaolegemitong/spark:1.0


克隆git仓库
git clone git@github.com:miaolegemitong/spark-docker.git


创建网络
sudo docker network create --driver=bridge hadoop


启动容器

这里需要把想要运行的jar包的路径作为第一个参数传递，之后这个路径会被挂载到master容器的/root/jars目录。

cd spark-docker
sudo ./start-container.sh <your jars path>



output:

start master container...
start slave1 container...
start slave2 container...
root@hadoop-master:~#
  • 启动了1个master、2个slave的hadoop集群
  • 启动了1个master、3个slave(master机器也作为slave使用）的Spark集群
  • 之后进入hadoop-master容器的/root目录

    启动Hadoop
    ./start-hadoop.sh

    运行Hadoop word count
    ./run-wordcount.sh


    output:
input file1.txt:
Hello Hadoop

input file2.txt:
Hello Docker

wordcount output:
Docker    1
Hadoop    1
Hello    2


启动Spark
./start-spark.sh


任意个数节点的集群


拉取docker镜像，克隆git仓库

重复3节点集群中的1-3步

重新build docker镜像
./resize-cluster.sh 5
  • 指定大于3的参数
  • 脚本将重写不同的slaves文件

启动容器
sudo ./start-container.sh <your jars path> 5
  • 使用和第2步相同的参数


启动Hadoop和Spark集群

和3节点中第5-7步一致
